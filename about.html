General Usage

This website serves a simple purpose: sort a Discogs record collection by the Notes field. Particularly, it looks at the Notes field, looks for a dollar value, and assigns the dollar value to a record. When all of the records have been assigned a dollar value, they are sorted by their values.

By default, the first 25 records are displayed in the table. The next page of records can be viewed by navigating to the bottom of the table and changing the value in the 'Page' field, then clicking submit. The number of records displayed per page can also be changed in the 'Per Page' field.

When the hompage is loaded, the collection will be (by default) sorted by the Notes field. i.e. The most valuable records will be listed first. The records can also be sorted by Date Added. This is accomplished by ticking the 'Sort by Date Added' box at the bottom of the table. Then click submit.


Updating the Collection

When you view your collection on lollookup.com, you are viewing static data (a snapshot). That is to say, when changes are made to your collection on discogs.com, those changes are not (automatically) realized on lollookup.com. In fact, no changes can be made to your Discogs record collection on this website. This website is just for viewing your collection in a way that Discogs has not implemented.

However, once you have made changes to your collection on discogs.com you can update this website's static data (take a new snapshot). This is accomplished by clicking the 'Update Collection' link at the top of the page.


Known Bugs and Caveats (Important)

- The preferred format for assigning a value to a record (when you are adding a new record on discogs.com) is as follows: $10 - Your Notes Here
The dollar sign ($) is important, and the dash (-) is important. Using this format will ensure that the note parser will read the correct value for the record.

- When updating your collection (adding new records) on discogs.com, please input a value for all of the record's fields. e.g. Assign values for: the Media Condition, the Sleeve Condition, and the Notes field.

- (Caveat that could be fixed in the future) When an update is made (Update Collection link is pressed), this website does not look for changes made to records that are already in its database. It only looks for new records that have been added to the collection on discogs.com. That is to say, once the record is added to this website's database, its Note field will never change, even if that record's Note field is changed on discogs.com . If changes have been made to the Discogs Record Collection (e.g. updated notes or removed particular records), the lollookup.com database can be updated (synced) by visiting this link. Warning: The Discogs devloper API only allows for 100 records to be requested per API call, meaning that (at the time of this writing) 13 API calls will be made when that link is visited. That will take some time, but if you wait you'll eventually be lead to a page that will display any changes that were made in order to sync the database to the Discogs record colleciton.
